 🔐 Secure AWS Architecture Project

I worked on a project to create a cloud architecture using Amazon Web Services. This included things like Virtual Private Cloud, public and private subnets, Elastic Compute Cloud and Simple Storage Service.

 Key Security Features:

. I set up a facing web server that can only be accessed through HTTP or HTTPS

. The private application server does not have an IP address

. The S3 bucket is secure with encryption. No one can access it from the outside

. I used Identity and Access Management roles that have the amount of privilege necessary

. I used a NAT Gateway to control what goes out to the internet

. Logging is turned on for things like CloudTrail and VPC Flow Logs

 Security Approach:

My approach to security was to use a combination of things to protect the Amazon Web Services architecture. This included keeping networks separate controlling who has access and encrypting data to reduce the risk of an attack and limit the damage if there is a breach. I used Amazon Web Services security features like Virtual Private Cloud and Identity and Access Management to make the architecture more secure. The goal was to make the Secure AWS Architecture Project as safe as possible by using defense-in-depth and reducing the risk of an attack, on the Secure AWS Architecture Project.
